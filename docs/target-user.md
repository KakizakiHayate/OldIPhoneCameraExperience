# ターゲットユーザー定義書

---

## 1. 基本属性

| 項目 | 内容 |
|------|------|
| 年齢 | 10代後半〜20代半ば（Z世代） |
| 性別 | 男女問わず |
| 特徴 | SNS（Instagram、TikTok）に写真を日常的に投稿する。レトロ・ヴィンテージな雰囲気を好む |
| デバイス | iPhone 12以降（iOS 16.0+） |
| 市場 | 日本市場を最初のターゲットとする |

---

## 2. ペルソナ例

> 22歳の大学生。Instagramのストーリーズやフィード投稿が日課で、写真の「雰囲気」にこだわりがある。
> 最近のiPhoneで撮る写真は「きれいすぎる」と感じていて、あえてフィルムカメラ風アプリや古いiPhoneで撮った写真の質感に惹かれている。
> TikTokで「iPhone 4で撮った写真がエモい」という投稿を見て、メルカリで古いiPhoneを探したこともあるが、バッテリー劣化や2台持ちの面倒さで断念した。
> 「最新iPhoneのまま、あの頃の撮影体験ができたらいいのに」と思っている。

---

## 3. ユーザーが抱える課題

### 3.1 最新iPhoneの写真への不満

- 計算写真（computational photography）による「きれいすぎる」写真に物足りなさを感じる
- HDR処理が強く、すべてが均一に明るく見えてしまい、雰囲気が出ない
- 友人と同じような写真になりやすく、個性が出しにくい

### 3.2 古いiPhone実機への憧れと障壁

- 古いiPhoneで撮った写真の「エモさ」に憧れるが、実機の入手にはハードルがある
  - メルカリ等で探す手間、バッテリー劣化のリスク、動作不安定
  - 2台持ち（最新iPhone + 古いiPhone）の煩わしさ
  - フィルムカメラほどではないが、コストと手間がかかる
- それでも「本物の古いデバイスで撮った」という体験と信憑性は、アプリでは得られないと感じている

### 3.3 既存フィルターアプリでは「何か違う」

- Dazz CamやHujiなど優秀なアプリはあるが、「加工した感」が残る
- 最新iPhoneの優秀なセンサーをベースにフィルターをかけるため、**古いセンサー特有の「根本的なぼんやり感」「処理の粗さ」が再現できていない**
- アプリは見た目の「エフェクト」は再現できるが、撮影の「体験」までは再現していない
- 「アプリで似た写真は撮れるけど、なんか違う…」という声が多い

---

## 4. 競合との差別化

### 「アプリ」vs「本物のデバイス」— 体験のレイヤーが根本的に違う

Z世代で古いiPhone実機での撮影が流行している理由は、フィルターアプリでは再現できない「体験そのもの」にある。

| 観点 | フィルターアプリ（Dazz等） | 古いiPhone実機 |
|------|--------------------------|---------------|
| 提供するもの | 見た目の「エフェクト」 | 時代を切り取る「体験」 |
| 写真の質感 | 最新センサーのクリアさ + フィルター → 「加工した感」 | 古いセンサーの根本的なぼんやり感・処理の粗さ → 「本物の不完全さ」 |
| SNSでの反応 | 「いいね、フィルターかわいい」 | 「マジで！？エモすぎ」「2台持ちかわいい」 |
| 信憑性 | 加工と分かりやすい | 本物のデバイスで撮った物語とオーセンティシティがある |
| 不完全さ | 便利すぎて不完全さが薄れる | ハードウェアの制限が本物の不完全さを生む |

**本アプリが目指す位置:** アプリの手軽さで、古いiPhone実機の「体験と本物感」を再現する。

### 競合比較

| 競合 | 特徴 | 本アプリとの違い |
|------|------|-----------------|
| **古いiPhone実機** | 本物の撮影体験。SNSでの信憑性が最も高い | 入手困難、バッテリー劣化、2台持ちの煩わしさ。本アプリは最新iPhoneのみで同じ体験を提供する |
| Dazz Camera | 多数のヴィンテージカメラモデルを再現。フィルム風フィルター多数 | 最新センサー + フィルターの構造のため「加工した感」が残る。見た目のエフェクトであり、撮影体験の再現ではない |
| Huji Cam | 使い捨てカメラ風。日付スタンプ、光漏れエフェクト | フィルムカメラの再現であり、iPhone 4の画質・体験の再現ではない |
| VSCO | 高品質なフィルター多数。編集機能が充実 | 撮影後の編集ツール。レトロな撮影体験を提供するアプリではない |
| Gudak Cam | フィルムカメラ風。撮影後に「現像待ち」の演出 | 使い捨てカメラの体験再現。iPhone特有の画質再現ではない |

### ポジショニング

```
  手軽（アプリ）──────────────────────────────────
        │                                │
  VSCO  │                                │  本アプリ
  Dazz  │  Huji                          │  （体験の再現）
        │  Gudak                         │
  ──────┼────────────────────────────────┼──────
        │                                │
        │                            iPhone 4実機
        │                            iPhone 5s実機
        │                          （入手困難・劣化リスク）
        │                                │
  見た目の加工（エフェクト）──── 体験の再現（本物感）
```

**本アプリの独自ポジション:** 「実機の入手不要で、古いiPhoneの"撮影体験そのもの"を再現するアプリ」

既存アプリが「見た目の加工」に留まる中、本アプリは「体験の再現」に踏み込むことで、古いiPhone実機の代替として唯一のポジションを取る。

## 5. 提供するコアバリュー

### 5.1 本物感のある撮影体験（最優先）

古いiPhone実機で撮ったかのような「体験」を、最新iPhoneで再現する。単なるフィルター（エフェクト）ではなく、古いセンサー特有の暖色系の色味、画角の狭さ、手ブレ補正がない時代の自然な揺れなど、**ハードウェアの制限そのもの**をシミュレートする。

> アプリは「エフェクト」だけど、本物は「体験全体」がレトロ。
> 本アプリは「体験全体」をアプリで再現することを目指す。

### 5.2 SNSでの信憑性（オーセンティシティ）

「これアプリで撮ったの？本物のiPhone 4？」と聞かれるレベルの再現度を目指す。写真に物語とオーセンティシティ（本物らしさ）を乗せることで、SNSでの話題性と差別化を実現する。

- 「Dazz Camで撮りました」→ 「いいね、フィルターかわいい」
- 「iPhone 4で撮りました」→ 「マジで！？エモすぎ」
- **本アプリで撮った写真が後者の反応を得られること**が目標

### 5.3 実機の障壁を解消

メルカリで古いiPhoneを探す手間、バッテリー劣化のリスク、2台持ちの煩わしさ——これらの障壁をすべて取り除き、最新iPhone1台で同じ撮影体験を提供する。

## 6. 利用シーン

| シーン | 具体例 |
|--------|--------|
| 日常のSNS投稿 | カフェ、街並み、友人との写真を「エモく」撮ってInstagramに投稿。「加工した感」がなく、自然なレトロ感 |
| 旅行・お出かけ | 旅先の風景を「記憶の中の色」で残す。不完全な写真が旅の空気感と「その瞬間の生々しさ」を伝える |
| 友人との撮り合い | 「このアプリで撮って！」と友人に頼まれる。撮影画面のレトロUIを見せること自体が話題になる |
| TikTok・Reelsコンテンツ | 撮影画面のスクリーンレコーディングがコンテンツに。「古いiPhone買わなくてもこれでいける」と紹介される |
| ノスタルジー体験 | iPhone 4世代を知っている人が懐かしさを楽しむ。知らない世代は「こんな写真が撮れたの？」と新鮮に感じる |

---

## 7. 成功の定義

ユーザーにとっての「成功体験」:

- 撮った写真をSNSに投稿して**「これiPhone 4で撮ったの？」と聞かれる**（最重要）
- 「アプリで撮ったの？全然加工っぽくない！」と驚かれる
- 古いiPhoneを買おうか迷っていた友人に「このアプリでいいじゃん」と教える
- アプリを開いた瞬間のレトロUIに「懐かしい！」「すごい！」と声が出る
- 友人にアプリを教えて、一緒に使い始める

## 8. Issue作成時の指針

機能追加や改善を提案する際は、以下のチェックリストで確認する:

| # | チェック項目 | 確認内容 |
|---|-------------|----------|
| 1 | **「体験の再現」に貢献するか？** | 見た目のエフェクト追加ではなく、古いiPhoneの撮影体験をより忠実に再現するか？ |
| 2 | **「本物感」を損なわないか？** | 便利すぎる機能を追加して「アプリっぽさ」が出ないか？不完全さのシミュレーションを壊さないか？ |
| 3 | **SNSでの信憑性に貢献するか？** | 撮影結果が「加工した感」ではなく「本物の古いiPhoneで撮った感」に近づくか？ |
| 4 | **実機の代替として強化されるか？** | 古いiPhone実機と比べたときの体験ギャップを埋める方向か？ |
| 5 | **ターゲットユーザーの課題を解決するか？** | Z世代のSNSユーザーが抱える「きれいすぎる写真」「実機入手困難」の課題を解消するか？ |

## 9. 技術的制約

ターゲットユーザーの体験に影響する技術的制約:

| 制約 | 内容 | ユーザー体験への影響 |
|------|------|---------------------|
| 対応機種 | iPhone 12以降（A14チップ以上） | リアルタイムフィルター処理にGPU性能が必要。古い機種は非対応 |
| 対応OS | iOS 16以上 | SwiftUI + Observation frameworkの要件 |
| カメラ許可 | カメラとフォトライブラリのアクセス許可が必須 | 初回起動時に許可ダイアログが表示される。拒否時はアラートで設定へ誘導 |
| アプリサイズ | 50MB以下 | シャッター音SE等のリソースを含めてもサイズを抑える |
| プレビュー性能 | 30fps以上 | フィルター処理が重いとプレビューがカクつき、体験が損なわれる |
| 撮影速度 | 撮影〜保存1秒以内 | 待ち時間が長いとテンポが悪く、連続撮影の楽しさが損なわれる |
| シャッター音 | 消音モードでも常に鳴る | 日本市場向け要件。静かな場所での撮影時にユーザーが認識しておく必要がある |
| 画面の向き | 縦持ち固定 | 横構図で撮影したい場合はトリミングが必要（Phase 2検討） |
| 再現の限界 | 最新センサーをベースにソフトウェアで古い画質をシミュレート | ハードウェアレベルの制限を完全に再現することは困難。チューニングと実機写真との比較テストが重要 |
